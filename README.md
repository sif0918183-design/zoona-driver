# ترحال زونا - مشروع دمج تطبيقات الراكب والسائق

هذا المشروع يدمج تطبيقي الراكب والسائق في مستودع واحد مع صفحة هبوط موحدة ونظام إشعارات مشترك.

## هيكل المشروع

- `landing.html`: صفحة الاختيار الرئيسية (تم تعيينها كصفحة الدخول في Vercel).
- `passenger_app/`: يحتوي على تطبيق الراكب.
- `driver_app/`: يحتوي على تطبيق السائق.
- `shared/`: يحتوي على ملفات الإعدادات والوظائف المشتركة.
- `schema.sql`: تعريف جدول قاعدة البيانات للإشعارات.

## تعليمات النشر

### 1. إعداد قاعدة البيانات
قم بتشغيل الكود الموجود في `schema.sql` داخل محرّر SQL في Supabase لإنشاء جدول `push_subscriptions`.

### 2. إعداد Vercel
تم ضبط ملف `vercel.json` لتوجيه جميع الطلبات إلى `landing.html` افتراضياً، مع استثناء مسارات التطبيقات والملفات المشتركة.

### 3. تحديث تطبيق Flutter (للسائقين)
إذا كنت تستخدم تطبيق Flutter WebView، يرجى تحديث `lib/main.dart` ليفتح صفحة السائق مباشرة أو صفحة الهبوط:

```dart
// لفتح تطبيق السائق مباشرة:
initialUrlRequest: URLRequest(url: WebUri('https://your-domain.com/driver_app/index.html')),

// أو لفتح صفحة الهبوط:
initialUrlRequest: URLRequest(url: WebUri('https://your-domain.com/landing.html')),
```

## نظام الإشعارات

يعتمد النظام على `Web Push API` و `VAPID`.
- المفتاح العام (VAPID Public Key) موجود في `shared/notification-utils.js`.
- يتم تخزين الاشتراكات في جدول `push_subscriptions`.
- يرسل تطبيق الراكب إشعارات عند تحديث حالة الرحلة.
- يرسل تطبيق السائق إشعارات عند وجود طلب رحلة جديد مع أزرار إجراءات (قبول/رفض).

## اختبار النظام

1. افتح النطاق الخاص بك.
2. اختر "تطبيق الراكب" أو "تطبيق السائق".
3. سيطلب المتصفح إذن الإشعارات، قم بالموافقة.
4. يمكنك استخدام Vercel Edge Functions أو Supabase Edge Functions لإرسال إشعارات تجريبية باستخدام `endpoint` المخزن في الجدول.
